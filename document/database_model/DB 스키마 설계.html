user_game_infomation(유저_게임_정보 매핑)

필드명설정설명
idauto_incrementpk테이블 pk
game_idfkvarchar(50)게임 아이디
uidfkvarchar(50)유저 아이디
is_joinedint관심게임 조인하기
is_ownedint사용자가 가진 게임
DB 테이블 템플릿
  • ManyToMany 관계인 유저와 게임 테이블을 매핑한다.
    • 한 유저는 여러 게임에 조인 할 수 있다.(is_joined)
    • 한 유저는 여러 게임을 구매했을 수 있다.(is_owned)
    • 한 게임에 관심을 가지는 여러 사람이 있다.
    • 한 게임을 구매한 여러 사람이 있다.
  • 유저가 joined 누른 게임과 steam_id를 등록한 사람에 한해서 가지고 있는 게임

user_game_recommend(유저_게임_추천 매핑)

필드명설정설명
idauto_incrementpk테이블 pk
game_idfkvarchar(50)게임 아이디
uidfkvarchar(50)유저 아이디
genre_idfkint장르별 추천을 위해서
scorefloat추천 스코어 (0~1 사이의 값)
DB 테이블 템플릿
  • 추천에서 삭제한 유저는 로그데이터에 쌓여 이후 추천에 뜨지 않게한다.
  • genre_id가 null인 경우 전체 게임데이터에서 Top-K개의 추천을, 장르 id가 있는 경우 해당 장르에서 Top-K개의 추천을 한다
  • 예를들어 장르에 구분없는 추천게임을 30개 장르별 추천게임을 10개, 장르의 수가 8개라고 가정하고 우리 서비스를 이용하는 유저의 수가 1000명일 경우 (30 + (10 * 8)) * 1000 = 110,000 개의 데이터가 이 테이블에 쌓여있고 스케쥴링 된 시간마다 이 데이터들이 갱신된다.

friend(친구 매핑 테이블)

필드명설정설명
idauto_incrementpk테이블 pk
from_uidvarchar(50)is_approved가 0인 경우 친구요청을 보낸 유저
to_uidvarchar(50)is_approved가 0인 경우 친구요청을 받는 유저
is_approvedint0 (승인대기), 1 (수락)
DB 테이블 템플릿
  • ManyToMany 관계인 친구(유저와 유저) 관계를 매핑한다.
  • 친구 요청 기능 - 수락을 해야만 서로 친구관계가 형성됨
# flow
A가 B에게 친구 신청을 하면 --> A(from_uid), B(to_uid), is_approved(0)
B가 수락을 하면 
				is_approved 변경 --> A(from_uid), B(to_uid), is_approved(1)
			  B와 A 친구 관계설정 --> B(from_uid), A(to_uid), is_approved(1)

DB 테이블 템플릿
